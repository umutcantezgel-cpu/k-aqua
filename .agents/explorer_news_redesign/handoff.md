# News Page Redesign Strategy (Milestone 2)

## 1. Observations
- **Target File**: `components/pages/NewsClient.tsx`
- **Current State**: The file is monolithic (247 lines). It primarily focuses on one major announcement (ISO 9001, 14001, 50001 certifications) and lacks the requested "10x expansion" for SEO depth.
- **Scroll Hijacking Violation**: It uses `useScroll` and `useTransform` (imported from `motion/react` on line 4) in a custom `ScrollSection` component (lines 186-246) to tie element opacity and vertical translation directly to scroll progress. This violates the `PROJECT.md` rule against scroll hijacking.
- **WebGL Mention**: Line 15 contains a comment `{/* GLOBAL BACKGROUND NOISE/WEBGL FOR AMBIANCE */}` which suggests previous or intended WebGL use, violating the NO WebGL rule.
- **Component Prefixing**: The page has not been broken down into `components/sections/` with the `News` prefix.
- **Theme Compliance**: Confirmed via grep that there are no `dark:` classes present; it adheres to the strict light theme.

## 2. Logic Chain
- To achieve a **10x content expansion for extreme SEO depth**, the page must shift from a single-announcement layout to a comprehensive News Hub. This requires distinct semantic sections targeting different search intents (e.g., long-form industry insights, press releases, company updates).
- To adhere to **Architecture Rules** (`PROJECT.md`), the monolithic structure must be split into isolated components in `components/sections/`, each strictly prefixed with `News` to prevent conflicts with parallel work on other pages.
- To eliminate **Scroll Hijacking**, the `useScroll`/`useTransform` dependencies must be purged. Animations should rely solely on simple `whileInView` or `initial`/`animate` triggers for subtle fade-ins, as mandated by the project guidelines.

## 3. Proposed Architecture & Component Plan

Create the following components in `components/sections/`:

1. **`NewsHeroSection.tsx`**
   - **Purpose**: The main entry point. Clean, large typography introducing the News & Insights hub.

2. **`NewsFeaturedArticleSection.tsx`**
   - **Purpose**: A deep-dive section retaining and expanding the existing ISO certification content (Quality, Environmental, Energy).
   - **SEO Focus**: High-value keywords related to "ISO certified manufacturing", "sustainable industrial processes".

3. **`NewsPressReleasesSection.tsx`**
   - **Purpose**: A chronological grid or list of recent press releases.
   - **Content**: E.g., "New Manufacturing Plant Opening", "Q3 Financial Growth", "Expansion into Middle East".

4. **`NewsIndustryInsightsSection.tsx`**
   - **Purpose**: **The primary SEO driver**. Long-form excerpts linking to detailed articles.
   - **Content**: Topics like "The Future of PPR Pipes in Sustainable Construction", "Water Security Innovations in 2026", "Decarbonizing Heavy Manufacturing".

5. **`NewsEventsExhibitionsSection.tsx`**
   - **Purpose**: Highlighting upcoming and past trade shows (e.g., ISH Frankfurt, Big 5 Dubai) to capture event-based search traffic.

6. **`NewsMediaKitSection.tsx`**
   - **Purpose**: Replaces the current "Verify Our Credentials" footer. Expands to include official brand assets, PR contacts, and high-res image downloads.

7. **`NewsNewsletterSignupSection.tsx`**
   - **Purpose**: Lead generation and audience capture.

*Note on Animations: All new sections must use a standard fade-in pattern:*
```tsx
<motion.div 
  initial={{ opacity: 0, y: 20 }} 
  whileInView={{ opacity: 1, y: 0 }} 
  viewport={{ once: true, margin: "-50px" }}
  transition={{ duration: 0.6, ease: "easeOut" }}
>
```

## 4. Caveats
- The actual textual content for the 10x expansion (the articles and press releases) needs to be generated by the implementing agent. This plan provides the structural skeleton.
- There is an existing `<MagneticButton />` component used in `NewsClient.tsx` that can be reused across the new sections.

## 5. Conclusion
The current `NewsClient.tsx` must be stripped of `useScroll`/`useTransform` and broken down into 7 specific `News`-prefixed section components. This structure will provide the vast surface area needed for the 10x SEO expansion while strictly adhering to the light-theme and anti-hijacking rules.

## 6. Verification Method
- **Component Prefix**: Run `ls components/sections | grep News` to ensure all new components are correctly prefixed.
- **No Scroll Hijacking**: Run `grep -r "useScroll" components/sections/News* components/pages/NewsClient.tsx` and expect NO results.
- **No Dark Mode**: Run `grep -r "dark:" components/sections/News* components/pages/NewsClient.tsx` and expect NO results.
- **No WebGL**: Ensure no `<canvas>` or WebGL-related terms exist in the output files.
