# Handoff Report: Home Page SEO Strategy & Component Breakdown

## 1. Observation
- `components/pages/HomeClient.tsx` currently contains ~220 lines of code with 4 hardcoded, inline sections: Hero, Kessel Partnership, Engineering Specifications, and a `HorizontalPanSection`.
- The `HorizontalPanSection` and Hero use `framer-motion`'s `useScroll` and `useTransform` for heavy scroll-linked animations (scroll hijacking).
- Content is currently sparse. It introduces K-Aqua but lacks the technical depth, product variety, and structured data necessary for competitive B2B SEO.
- `PROJECT.md` mandates breaking content into `components/sections/`, prefixing components with the page name (e.g., `HomeHeroSection.tsx`), strictly enforcing a light/bright theme (no dark mode), and avoiding extreme scroll hijacking in favor of simple `motion.div` fade-ins.
- `SCOPE.md` demands a 10x expansion of the Home Page content.

## 2. Logic Chain
1. **SEO Expansion (10x)**: To achieve massive SEO depth, the homepage cannot be a brief visual flyer. It must become a comprehensive pillar page. Search engines reward topical authority. We must expand the topics from just "We make pipes" to include Applications, Materials Science, Certifications, Sustainability, and a dedicated FAQ for long-tail "People Also Ask" queries.
2. **Component Architecture**: Following `PROJECT.md`, the monolithic `HomeClient.tsx` must be converted into a lean wrapper that imports dedicated section components from `components/sections/`. This prevents merge conflicts and enforces semantic isolation.
3. **Naming Convention**: Every new file must be prefixed with `Home` to avoid namespace collisions with future About or Contact sections.
4. **Motion & UX**: The `HorizontalPanSection` violates the "NO extreme scroll hijacking" rule and creates horizontal scrolling which can be detrimental to basic mobile SEO and accessibility if not handled perfectly. It must be replaced by a standard vertical layout utilizing simple `whileInView` fade-ins (`opacity: 0` to `opacity: 1`).
5. **Theming**: The strict light theme means all sections should oscillate between `bg-white` and subtle off-whites like `bg-slate-50`, using `text-slate-900` for headings and `text-slate-600` for body copy. `dark:` classes are strictly prohibited.

## 4. Conclusion
The implementer agent must gut `HomeClient.tsx` and replace it with a series of imports. I propose the following 10-section architecture to achieve the 10x SEO depth requirement:

1. **`HomeHeroSection.tsx`**: Core value proposition, H1 ("Leading in Water Supply"), and primary CTAs.
2. **`HomeFeaturesSection.tsx`**: High-level advantages (Precision Manufacturing, Material Superiority, Global Deployment).
3. **`HomeProductEcosystemSection.tsx`**: SEO keyword injection for specific product categories (e.g., PP-R Pipes, Fittings, Valves, Distribution manifolds).
4. **`HomeApplicationsSection.tsx`**: Target industries (Residential, Commercial, Industrial, Municipal) to capture sector-specific search intent.
5. **`HomePartnershipSection.tsx`**: The KESSEL alliance details for drainage synergies.
6. **`HomeMaterialsScienceSection.tsx`**: Technical specifications (polymers, thermal stability, biological neutrality) targeting engineering keywords.
7. **`HomeCertificationsSection.tsx`**: Trust signals (ISO, DIN, WRAS) crucial for B2B trust and technical SEO compliance.
8. **`HomeSustainabilitySection.tsx`**: Green building standards, recyclable plastics, and energy-efficient manufacturing keywords.
9. **`HomeFAQSection.tsx`**: Semantic HTML accordion addressing long-tail queries (e.g., "What are the benefits of PP-R pipes?", "How long do KWT fittings last?").
10. **`HomeCTASection.tsx`**: Final conversion block, replacing the old `HorizontalPanSection` with a fast-loading, vertically scrolling CTA.

**Styling Rules for Implementer**:
- Use `<motion.div initial={{ opacity: 0, y: 20 }} whileInView={{ opacity: 1, y: 0 }} viewport={{ once: true }} transition={{ duration: 0.6 }}>` for all entry animations.
- Use `bg-white`, `bg-slate-50`, `text-slate-900`, `text-slate-600`.
- Remove all imports of `useScroll` and `useTransform` from the home components.
