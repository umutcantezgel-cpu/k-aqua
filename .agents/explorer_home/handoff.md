# Handoff: Home Page Expansion Strategy

## 1. Observation
- `components/pages/HomeClient.tsx` currently contains four main inline sections:
  1. Hero section with parallax/scroll-linked opacity effects.
  2. "Partnership & Experience" section with scroll-linked translations.
  3. "Engineering Specifications" section with simple `whileInView` fade-ins.
  4. "Horizontal Pan / Deep Dive" section which uses extreme scroll hijacking to pan horizontally.
- The user requirements explicitly state:
  - 10x content expansion for extreme SEO depth.
  - Break down into `components/sections/` with `Home` prefix.
  - Strictly light, bright, professional theme (NO dark mode classes).
  - NO WebGL or extreme scroll hijacking (replace with simple `motion.div` fade-ins).
- `PROJECT.md` confirms Next.js 15 App Router, Tailwind CSS, and simple fade-ins.
- `.agents/sub_orch_m2/SCOPE.md` specifies this as Milestone 1 of Pages Batch 1.

## 2. Logic Chain
- To achieve a 10x expansion for extreme SEO depth, the page must cover a broader semantic footprint related to water supply, manufacturing, materials, and B2B plumbing solutions. We must expand beyond the current brief summary.
- The scroll hijacking in `HorizontalPanSection` violates the "NO extreme scroll hijacking" constraint and must be replaced with a standard vertically-scrolling CTA section.
- The complex `useScroll` and `useTransform` hooks for hero and partnership sections should be simplified to use basic `motion.div` `initial` and `whileInView` props to comply with the "simple motion.div fade-ins" rule.
- To prevent component naming collisions across parallel implementations (as per `PROJECT.md`), all new components will be prefixed with `Home`.
- I have formulated a 10-section structure that targets specific SEO content silos (Products, Applications, Sustainability, FAQ) while maintaining the professional, light aesthetic.

## 3. Caveats
- I am operating in a read-only capacity; I have not created the files in `components/sections/` nor updated `HomeClient.tsx`.
- The exact copy (text) for the new sections (e.g., FAQ questions, product categories) will need to be generated by the implementer or provided via a content system.

## 4. Conclusion
**Proposed Section Breakdown (10 Sections):**

1. `HomeHeroSection.tsx`: Simplified hero without complex scroll hooks. Focus on "Leading in Water Supply" and value proposition.
2. `HomeCertificationsSection.tsx`: *NEW.* A logo cloud or badge row displaying ISO certifications, global standards, and KESSEL partnership to build immediate B2B trust.
3. `HomePartnershipSection.tsx`: Adapted from the current Section 2. Details the KESSEL partnership, decades of experience, and unified liquid transport ecosystem.
4. `HomeEngineeringSpecsSection.tsx`: Adapted from the current Section 3. Precision manufacturing, material superiority.
5. `HomeProductCategoriesSection.tsx`: *NEW.* High-level overview of product lines (Pipes, Fittings, Valves, Accessories) with internal links. Critical for SEO siloing.
6. `HomeApplicationsSection.tsx`: *NEW.* Outlines use cases (Commercial, Industrial, Residential, Extreme Environments). Targets long-tail keywords for specific industries.
7. `HomeSustainabilitySection.tsx`: *NEW.* Focus on eco-friendly polymers, recyclability, and biological neutrality. Targets "sustainable water transport" keywords.
8. `HomeGlobalReachSection.tsx`: *NEW.* Maps or stats showing worldwide deployment and logistics reliability.
9. `HomeFAQSection.tsx`: *NEW.* An accordion-style FAQ answering common B2B engineering questions (e.g., polymer vs. metal, temperature limits). Excellent for Schema markup and deep SEO.
10. `HomeCTASection.tsx`: Replaces the horizontal pan. A clean, high-impact call to action ("Initiate Project") with a simple fade-in.

**Implementation Directives:**
- All files to be created in `components/sections/`.
- `components/pages/HomeClient.tsx` will be refactored to simply import and stack these 10 components.
- Use `initial={{ opacity: 0, y: 20 }} whileInView={{ opacity: 1, y: 0 }}` for animations.
- Ensure all Tailwind classes use light-mode only (e.g., `bg-white`, `bg-slate-50`, `text-slate-900`). Remove any existing `dark:` classes if found.

## 5. Verification Method
- **File inspection**: Check `components/pages/HomeClient.tsx` to verify it imports the 10 `Home`-prefixed components and contains no scroll hijacking logic.
- **Component inspection**: Verify the existence of `components/sections/Home*.tsx` files. Check for the absence of `dark:` classes in these files using `grep_search`.
- **Run project**: Run `npm run build` to ensure no component import errors exist.
