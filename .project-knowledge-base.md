# Project Knowledge Base

## [SECTION:GOTCHAS]

- **React Hook Order**: Hooks wie `React.useId()` dürfen auf keinen Fall kurzgeschlossen oder conditionally via `||` Operators evaluiert werden. Dies triggert harte `rules-of-hooks` Fehler (TYPE-D/RUNTIME FATAL).
- **ESLint State in Effect**: Der Linter erzwingt streng die `react-hooks/set-state-in-effect` Regel. Wenn `useEffect` zwingend notwendig ist (z.B. für `isClient`-Hydration-Workarounds oder Mount-Flags), **muss** ein gut begründeter `// eslint-disable-next-line react-hooks/set-state-in-effect` Kommentar eingefügt werden.
